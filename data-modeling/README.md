# Data Modeling

This folder contains learnings about schema design, partitioning strategies, and data organization in Azure Cosmos DB.

## Topics Covered

- Document structure patterns
- Embedding vs. referencing strategies
- Hierarchical Partition Keys (HPK)
- Hot partition prevention
- Denormalization patterns
- Change feed design considerations
- Time-series data modeling
- Multi-tenancy patterns
- Data lifecycle management

## Document Template

When adding a new learning, consider including:

1. **Title**: Clear, descriptive title
2. **Use Case**: What was the application or scenario?
3. **Initial Design**: How was data initially modeled?
4. **Challenges**: What issues arose from the initial design?
5. **Revised Design**: How was the model improved?
6. **Schema Examples**: JSON document examples
7. **Query Patterns**: How data is accessed
8. **Lessons Learned**: Key takeaways and recommendations
